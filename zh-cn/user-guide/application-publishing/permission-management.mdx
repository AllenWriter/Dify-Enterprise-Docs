---
title: 发布权限管理
---

> 仅 Workspace Owner、Workspace Admin 具备创建和发布应用的权限。关于各成员的权限说明，详细说明请参考[成员权限管理](/zh-cn/administrator-guide/members-management#成员权限管理)。

创建应用后，默认情况下该应用不对任何人开放。为确保数据安全，建议添加访问权限后再开放应用。若未配置应用权限，将无法生成应用的访问链接、获取应用嵌入代码。

本文将介绍如何配置和管理应用的访问权限。

## 为应用添加访问权限

1. 在应用编辑界面，点击右上角的 **Publish** 按钮。
2. 在发布页内的 **Who can access web app** 选项中进行权限配置。

![](https://assets-docs.dify.ai/2025/04/620c671984aa3a8c6c874af6d8ddaec7.png)

## 设置应用访问权限

Dify 提供了三种访问权限模式，你可以根据应用的使用场景选择合适的权限：

### 仅组织内成员访问

> 选择该权限选项前，请确保已开启 WebApp SSO。详细配置方法请参考[认证方式 - WebApp 配置](/zh-cn/administrator-guide/authentication#webapp-配置)。

该权限仅限团队内的成员访问应用，通常适用于面向企业内部全体员工的应用。选择该权限后，其他成员在访问在线应用链接时，要求登录企业预设的 SSO 服务。

### 特定群组或成员访问

该权限选项限制应用仅对团队中特定群组的成员开放，帮助开发者精确控制应用的访问权限。

**配置方法：**

- 允许特定群组访问

点击 **Add** 按钮添加特定群组，该群组内的全体成员将自动获取应用的访问权限。若群组内新增或移除了成员，将自动授予或撤销该成员的应用访问权限。

![](https://assets-docs.dify.ai/2025/04/02f1c655d6f70562f56c5b4c58fb436b.png)

- 允许特定成员访问

选择分组后，点击右侧的扩展按钮选择群组内的成员。被选中的成员将始终具备应用的访问权限，不受群组内其他成员变动的影响。相同群组内未被选中的成员则无法访问应用。

![](https://assets-docs.dify.ai/2025/04/8f456f2e5c3575a5d1be5fa75d5475a4.png)

> 当选择特定群组或成员访问但尚未添加任何群组或成员时，系统会显示 **No access members have been added yet**（尚未添加访问成员）的提示，此时应用处于无人可访问状态。

<Info>
    Workspace Owner 和 Workspace Admin 拥有访问和编辑 Workspace 内所有应用的权限。但需注意，若该 Owner 或 Admin 未被添加到 Web App 的访问权限列表中，他们依然无法调整该应用的发布权限。
</Info>

### 任何持有链接的人访问

该选项将允许任何获得链接的人访问应用，通常适用于公开演示的应用、面向客户的服务或允许公开获取的资源。

## 常见问题

### 是否需要重新发布应用才能使权限变更生效？

不需要。权限变更会立即生效，无需重新发布应用。但请注意，已获得访问权限的成员当前会话不会立即失效，可能需要等待会话过期后才会应用新的权限设置。

### 如何确保只有特定成员才能访问我的应用？

你可以选择"特定群组或成员访问"选项，然后添加需要访问的群组或具体成员。这样只有被添加的群组中的成员或被明确指定的个人才能访问你的应用。

### Workspace Owner 或 Admin 是否自动拥有所有应用的访问权限？

Workspace Owner 和 Admin 虽然拥有访问和编辑 Workspace 内所有应用的权限，但如果他们未被添加到应用的访问权限列表中，将无法调整该应用的发布权限和使用权限。

### 为什么我无法生成应用的访问链接或嵌入代码？

如果你尚未配置应用的访问权限，系统将无法生成应用的访问链接或获取嵌入代码。请先完成权限配置后再尝试获取链接或嵌入代码。

### 如何查看谁有权限访问我的应用？

你可以在应用的发布页面中的"Who can access web app"（谁可以访问网页应用）选项查看当前拥有访问权限的群组和成员列表。

### API 密钥权限与 Web 应用权限有什么区别？

API 密钥权限与 Web 应用权限是分开管理的。更改 Web 应用的访问权限不会影响通过 API 密钥访问应用的权限设置。
