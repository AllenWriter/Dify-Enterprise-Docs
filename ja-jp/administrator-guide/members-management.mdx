---
title: メンバー管理
---

システム管理人は Members ページで現在の企業内のすべてのメンバーを表示および管理できます。これには、各 Workspace の権限ロール、メンバーの状態、メンバーの作成日および最終アクティブ日の表示が含まれます。

![](/ja-jp/images/members-management-1.png)

## メンバー権限管理

各メンバーにはチームレベルと Workspace レベルの2つの権限があります。チーム管理人は一般メンバーを追加し、新しいシステム管理人を招待することができます。また、特定のメンバーに Workspace の権限を割り当てることもできます。

![](https://assets-docs.dify.ai/2025/04/60a427910a4f698d4b62e4aa0098ff55.png)

### チームレベルの権限

- 一般ユーザー
Members ページで追加されたユーザーはデフォルトで一般ユーザー権限を持っています。
![](/ja-jp/images/members-management-3.png)
  このユーザーグループに属するメンバーは以下の操作を行うことができます：
  - Dify 企業版にログインして使用する

- システム管理人
Settings ページで追加されたユーザーはデフォルトでシステム管理人権限を持っています。システム管理人の操作ガイドについては、[システム設定](/administrator-guide/system-settings)を参照してください。
![](/ja-jp/images/members-management-4.png)

  このユーザーグループに属するメンバーは以下の操作を行うことができます：
  - 企業版管理バックエンドにログインする
  - チームメンバーを追加/削除/無効化する
  - Workspace にメンバーを追加し、権限を割り当てる

### Workspace レベルの権限

Workspace には Owner、Admin、Normal の3つの権限グループがあります。
![](/ja-jp/images/members-management-5.png)

- Owner
Workspace の作成者であり、各 Workspace には1人の Owner しかいません。Workspace 内のすべてのアプリケーションの作成および編集権限を持っています。設定ページにアクセスしてモデルプロバイダーを追加したり、API 拡張を作成したりすることができます。
- Admin
Workspace の管理人であり、各 Workspace には複数の管理人が存在することができます。Workspace 内のすべてのアプリケーションの作成および編集権限を持っています。設定ページにアクセスしてモデルプロバイダーを追加したり、API 拡張を作成したりすることができます。
- Normal
Workspace の一般メンバーであり、各 Workspace には複数の一般メンバーが存在することができます。Workspace 内のアプリケーションを使用することはできますが、編集や変更はできません。設定ページにアクセスしてモデルプロバイダーを追加したり、API 拡張を作成したりすることはできません。
- Workspace Editor

Workspace 内のアプリケーションの作成および編集権限を持っています。設定ページにアクセスしてモデルプロバイダーを追加したり、API 拡張を作成したりすることはできません。

- Knowledge Admin
Workspace 内の知識ベースの作成および編集権限を持っています。アプリケーションの作成および編集はできません。設定ページにアクセスしてモデルプロバイダーを追加したり、API 拡張を作成したりすることはできません。

### チームに新しいメンバーを追加する

チームが SSO 認証を有効にしていない場合、ユーザー名とパスワードを使用してログインできます。
右上の青い Add ボタンをクリックし、ユーザー名とメールアドレスを入力します。メンバーを初めて追加する際には、デフォルトのログインパスワードが表示されます。チームに管理人を追加する方法については、[システム管理人設定](/administrator-guide/system-settings#)を参照してください。
![](/ja-jp/images/members-management-6.png)

### Workspace に新しいメンバーを追加する

特定の Workspace に新しいメンバーを追加したい場合は、[Workspace 管理](/administrator-guide/workspace-management)を参照してください。

### メンバーを無効にする

メンバーを直接削除するのではなく、無効にすることでデータを保持しながらユーザーのアクセス要求を拒否することができます。メンバーの右側のメニューボタンをクリックし、Status を banned 状態に切り替えることでメンバーを無効にできます。
![](/ja-jp/images/members-management-7.png)

### メンバーのパスワードをリセットする

メンバーがパスワードを紛失した場合、システム管理人が手動でパスワードリセットメールを送信する必要があります。メンバーの右側のメニューボタンをクリックし、Reset Password を軽くタップします。

![](/ja-jp/images/members-management-8.png)

### メンバーを削除する

⚠️ メンバーのすべてのアプリケーションと使用データが削除されるため、慎重に処理してください。
メンバーの右側のメニューボタンをクリックし、"Delete" オプションを軽くタップすることでメンバーを削除できます。

![](/ja-jp/images/members-management-9.png)

Workspace のメンバーは一度追加されると削除できません。Members ページの Delete をクリックすると、チーム内のメンバーが削除されます。

## メンバーを検索する

管理人は Workspace ID とユーザーのメールアドレスを使用して検索ボックス内でユーザーを検索できます。また、ユーザーの状態を切り替えて検索することもできます。

![](/ja-jp/images/members-management-10.png)

## メンバーグループ管理

<Info>
  この機能は Dify エンタープライズ版 バージョン ≥v2.6.0 が必要です。
</Info>

Dify エンタープライズ版では、**グループ管理** はメンバーが社内のさまざまなリソースへアクセスする権限を制御する仕組みです。メンバーを異なるグループに分けることで、異なるユーザーが異なる Web アプリにアクセスできるようになり、リソースの誤用や潜在的なデータ漏洩リスクを回避し、企業情報の安全を守ることができます。

### グループの作成

**メンバー** ページ右上の **グループ管理** をクリックし、グループ名を入力して作成します。グループ名は重複できません。

![](https://assets-docs.dify.ai/2025/04/cb14986e1d2346aff6c5e321ced7cff7.png)

### グループの編集

グループは名前の変更やドラッグ＆ドロップによる並び替えが可能です。グループ名の右側にある **編集** ボタンをクリックして名前を変更できます。名前は既存のグループと重複できません。

### ユーザーの割り当て

メンバーの右側にある **メニュー** ボタン → **グループ編集** をクリックし、参加させたいグループを選択します。1人のユーザーが複数の異なるグループに同時に所属することができます。

![](https://assets-docs.dify.ai/2025/04/2e20d0adc2cbeb1267b6b1639c1ec677.png)

ユーザーにグループを追加した後は、[アプリ公開：アプリのアクセス権限管理](zh-cn/user-guide/application-publishing/permission-management) を参照して Web アプリのアクセス権限を管理してください。
