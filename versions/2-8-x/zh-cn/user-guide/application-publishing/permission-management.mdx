title: Web 应用访问权限控制
---

<Info>
  仅工作空间内的 Owner、Admin、Editor 角色具备创建和发布 Web 应用的权限。关于各成员的权限，详细请参考[成员权限管理](/zh-cn/administrator-guide/members-management#成员权限管理)。
</Info>

创建 Web 应用后，默认情况下 Web 应用的可访问权限是“平台内指定成员”。若范围留空，表示不对任何成员开放。本文将介绍 Web 应用的访问权限。

## 修改 Web 应用访问权限

通过以下入口修改 Web 应用访问权限：

1. 访问 Studio 页，点击应用右下角菜单内的 Web 应用访问权限。
2. 进入 Web 应用编辑界面，点击右上角的 **Publish** 按钮。在 **Who can access web app** 区域内为 Web 应用设置访问权限。

![](https://assets-docs.dify.ai/2025/06/42265a0a3956490c68173ea2800d5505.png)

## 访问权限详情

Web App 提供四种访问权限：

![](https://assets-docs.dify.ai/2025/06/4f96327ddf919998a45c095511fae116.png)

### 平台内所有成员

> 面向内部用户，即 Dify 企业版内的成员。用户访问 Web App 链接时要求需通过内部用户身份认证。详细配置说明请参考[内部用户认证](/versions/2-8-x/zh-cn/administrator-guide/authentication/enterprise-user-auth)。

该权限仅限已加入 Dify 企业版内的成员访问 Web 应用。选择该权限后，访问 Web 应用链接时将弹出 Dify 平台的登录验证页。用户可使用账号和密码、账号和验证码或 SSO 认证，通过后即可访问 Web 应用。

> 若在 Dify 企业版 ＜ 2.8.x 的版本中曾开启 Web App SSO 设置，升级至 v2.8.x 后将当前的 Web App 的访问权限范围将自动被设置为"经认证的外部用户"。

启用该选项后，用户可通过访问 **Web App URL** 或在工作空间的 **Explorer** 页中访问 Web 应用。

### 平台内指定成员

> 面向内部用户，即 Dify 企业版内的成员。用户访问 Web App 链接时要求需通过内部用户身份认证。详细配置说明请参考[内部用户认证](/versions/2-8-x/zh-cn/administrator-guide/authentication/enterprise-user-auth)。

创建新 Web App 时的默认权限选项，限制 Web App 仅对团队中特定群组的成员开放。例如由“销售部门”所创建的，带有敏感销售数据的 Web App 仅面向销售部门内的成员开放。

若选择该权限项但尚未添加任何群组或成员时，Web App 处于无人可访问状态，无法生成应用的访问链接与嵌入代码。

**配置方法：**

- **允许特定群组访问**

点击 **Add** 按钮添加特定群组，该群组内的全体成员将自动获取 Web 应用的访问权限。当群组内新增成员时，系统将自动授予该成员Web 应用访问权限；当成员被从群组中移除时，系统将自动撤销其Web 应用访问权限。

![](https://assets-docs.dify.ai/2025/06/2ae6a255c949c0e28ab2acd087db9b62.png)

- **允许特定成员访问**

被选中的成员将始终具备Web 应用的访问权限，即便被移除出群组也具备Web 应用访问权限；同群组内未被选中的成员无法访问Web 应用。

> 当选择特定群组或成员访问但尚未添加任何群组或成员时，Web 应用处于无人可访问状态，并且无法生成Web 应用的访问链接、获取Web 应用嵌入代码等操作。

<Info>
    Workspace Owner、Admin 和 Editor 拥有访问和编辑 Workspace 内所有Web 应用的权限。但需注意，若该 Owner 或 Admin 未被添加到Web 应用的“平台内指定成员”列表内，他们依然无法通过在线链接访问Web 应用。
</Info>

### 经认证的外部用户

> 面向外部用户，即 Dify 企业版外的成员。如需了解如何配置面向外部成员的 SSO 配置，请参考 [Web App 外部用户认证](/versions/2-8-x/zh-cn/administrator-guide/authentication/webapp-external-user-auth)。

选择该选项后，Dify 企业版平台外部的用户在访问 Web App 链接时将被要求需通过 SSO 认证。企业内的管理人员可通过第三方身份提供商（IdP）统一监测和管理外部用户，与 Dify 企业版内的成员数据相隔离。

**常见Web 应用场景：**
- 合作伙伴门户：为供应商提供专用的业务Web 应用
- 产品咨询平台：为潜在客户提供产品介绍和技术支持

<Info>
  若发现该选项右侧出现禁用提醒，请联系系统管理员前往管理后台，查看是否已配置并开启 Web App 外部用户 SSO 认证。详细说明请参考 [Web App 外部用户认证](/versions/2-8-x/zh-cn/administrator-guide/authentication/webapp-external-user-auth)。
</Info>

### 任何人

该选项将允许任何获得互联网链接的用户访问Web 应用，无需认证流程。通常适用于公开演示的Web 应用、面向客户的服务或公开资源。

## 常见问题

### 是否需要重新发布Web 应用才能使权限变更生效？

不需要。权限变更会立即生效，无需重新发布Web 应用。但请注意，已获得访问权限的成员当前会话不会立即失效，可能需要等待会话过期后才会Web 应用新的权限设置。

### 如何确保只有特定成员才能访问我的Web 应用？

你可以选择"特定群组或成员"选项，然后添加需要授予访问权限的成员。

### 如何查看谁有权限访问我的Web 应用？

你可以在Web 应用的发布页面中的 **Who can access web app（谁可以访问 Web Web 应用）** 选项查看拥有当前Web 应用访问权限的群组和成员列表。

### 如何选择合适的权限模式？

建议按照以下原则选择：

- **内部协作工具**：选择“平台内所有成员”
- **部门专用Web 应用**：选择“平台内指定成员”
- **客户服务Web 应用**：选择“经认证的外部用户”
- **公开演示**：选择“任何人”（请谨慎使用）

### 调整Web 应用访问权限后，是否会影响 API 调用？

不会。Web 应用的 API 访问权限由 API Key 独立控制，与 Web Web 应用的访问权限无关。

只要确保 API Key 保持有效且未泄露，就可以正常通过 API 调用Web 应用。即使更改了 Web Web 应用的访问权限设置，也不会影响已有 API Key 的调用权限。
