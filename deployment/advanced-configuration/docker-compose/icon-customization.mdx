通过替换默认资源文件来自定义 Web 界面。本指南将演示如何替换关键视觉元素并启动带有自定义内容的应用程序。

## 前提条件

- 已安装 Docker 和 Docker Compose
- 准备好用于替换的自定义视觉文件

## 自定义步骤

1. **替换资源文件**
   在 `web_assets` 目录中添加以下自定义版本的文件：
   - `logo-site.png` 主导航栏 logo
   - `favicon.ico` 浏览器标签图标

2. **使用自定义资源启动**
   使用基础配置和 web 资源配置一起启动应用程序：

   ```bash
   docker compose -f docker-compose.yaml -f docker-compose.with-web-assets.yaml up -d
   ```

## 文件规格

| 文件            | 用途                    | 推荐规格                   |
| --------------- | ----------------------- | -------------------------- |
| `logo-site.png` | 左上角导航栏 logo      | PNG 格式                   |
| `favicon.ico`   | 浏览器标签图标         | ICO 格式，48×48 像素       |

## 重要说明

- 🔄 **需要重启**：修改资源文件后，需要重启容器才能查看更改。
- 🖼️ **资源要求**：保持文件名和格式完全一致，以确保正确识别。

## 目录结构示例

```bash
project/
├── web_assets/
│   ├── logo-site.png    # 你的自定义 logo
│   └── favicon.ico      # 你的自定义网站图标
├── docker-compose.yaml
└── docker-compose.with-web-assets.yaml
```
